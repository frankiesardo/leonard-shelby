# Affordances and Signifiers

An affordance is a relationship between an object and the person who interacts with it -- it describes what actions are possible. A flat surface affords support; a button affords pushing; a slot affords inserting. Affordances exist whether or not you perceive them. A hidden button still affords pushing, but it's useless if nobody finds it.

This is where signifiers come in. A signifier is any perceivable indicator that communicates what action is possible and how to do it. A "push" sign on a door is a signifier. So is a flat plate that your hand naturally presses against. The plate is better because it doesn't require reading -- the affordance and the signifier are fused. Norman's original sin was conflating affordances and signifiers in the first edition of DOET. The revised edition corrects this: affordances define what's possible; signifiers communicate what's possible. The distinction matters because designers can't always control affordances (the physics of the object), but they can always control signifiers (the communication layer).

The practical test is simple: if a door needs a sign that says "push," the design has failed. The hardware itself should communicate the action. A flat plate says push. A handle says pull. When the hardware says pull but the door must be pushed, you get Norman doors -- the universal emblem of bad design.

## Why it matters

The affordance-signifier framework applies far beyond physical objects. Every software interface, every API, every form, every set of instructions is an exercise in making affordances perceivable through signifiers. The reason so much technology is frustrating is not that it lacks capability (affordances) but that it fails to communicate that capability (signifiers). Features that users can't discover might as well not exist.

For product builders, the discipline is: don't just build the capability. Build the communication of the capability into the object itself. If you need a manual to explain it, you haven't designed it -- you've documented your failure.
