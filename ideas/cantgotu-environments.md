# Cantgotu Environments

Not everything that is logically possible is physically possible. The overwhelming majority of logically possible environments cannot be rendered by any physically possible virtual-reality generator. Deutsch calls these "Cantgotu environments" -- a portmanteau of Cantor, Godel, and Turing, the three mathematicians whose work establishes the proof, and a pun: environments you *can't go to*.

The proof uses Turing's diagonal argument. List every possible program for a universal computer. Each generates an environment. Now define a new environment that differs from Environment 1 during minute 1, from Environment 2 during minute 2, and so on. This new environment is not on the list -- it cannot be generated by any program. Since the list was exhaustive, no physically possible machine can render it. And there are infinitely many such environments for every one that is renderable.

What would a Cantgotu environment feel like? You would keep trying to guess which program was generating it, and you would always eventually be proved wrong. Not because the environment was tricky, but because *no* program generates it.

## Why it matters

The existence of Cantgotu environments establishes an absolute boundary on knowledge and computation. It is not a limitation of current technology -- it is a consequence of the laws of physics. Some questions are non-computable: they have definite answers, but no physical process can produce those answers. This is humbling. Even the omega-point intelligence at the end of the universe, with unbounded computational resources, cannot access the overwhelming majority of logically possible truths.

But the boundary also clarifies what *is* possible. Within the boundary, the Turing principle guarantees that everything physically possible is renderable. The computable world is vast and self-similar. The non-computable world is vaster still, but permanently inaccessible. The structure of this boundary -- what is on which side -- is itself a deep fact about the fabric of reality.
